API ENDPOINTS (for reference):

POST   /api/auth/login              → User login
GET    /api/products                → Get all products
GET    /api/products/category/:cat  → Filter by category
POST   /api/orders                  → Create order
GET    /api/orders                  → View all orders (staff)
PUT    /api/orders/:id              → Update order status
DELETE /api/orders/:id              → Delete order (staff)

Full documentation: See API_DOCS.md

🐛 IF SOMETHING DOESN'T WORK:

1. Is backend running?
   → PowerShell should show "Server is running"

2. Port 5000 already in use?
   → Check: netstat -ano | findstr :5000
   → Kill process or use different port

3. Products not loading?
   → Check browser console (F12)
   → Verify app-backend.js is linked in HTML

4. Login not working?
   → Check localStorage (F12 → Application)
   → Verify form elements have correct IDs

5. CORS errors?
   → Make sure backend is http://localhost:5000
   → Check frontend is calling /api endpoints

More help: See TESTING.md or MIGRATION.md
